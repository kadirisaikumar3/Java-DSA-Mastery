# Dynamic Programming Patterns

Dynamic Programming (DP) solves complex problems by storing solutions to overlapping subproblems.

Instead of recomputing results, DP reuses previously calculated values.

---

## Common DP Patterns

- Fibonacci
- 0/1 Knapsack
- Unbounded Knapsack
- Longest Common Subsequence
- Longest Increasing Subsequence
- Matrix DP
- Partition DP
- Interval DP

---

## DP Approaches

### Memoization

Top-down recursion with caching.

### Tabulation

Bottom-up iterative solution.

---

## Common Problems

- House Robber
- Coin Change
- Decode Ways
- Edit Distance
- Target Sum
- Climbing Stairs

---

## Complexity

Depends on the state definition but typically:

| Time | O(n²) or better |
|------|-----------------|
| Space | O(n²), O(n), or optimized |

---

## Key Idea

Define states, transitions, and base cases before implementing the solution.
