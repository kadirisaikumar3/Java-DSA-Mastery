# Interview Strategy

Mastering coding interview patterns is only one part of interview success.

Top companies evaluate:

- Problem Solving
- Communication
- Code Quality
- Optimization Skills
- Testing Ability

---

## Step 1 — Understand the Problem

Before coding:

- Read carefully
- Clarify constraints
- Ask questions
- Identify edge cases

---

## Step 2 — Identify the Pattern

Common patterns include:

- Two Pointers
- Sliding Window
- Binary Search
- DFS/BFS
- Dynamic Programming
- Greedy
- Union Find
- Backtracking
- Heap
- Monotonic Stack

Choosing the correct pattern simplifies the solution significantly.

---

## Step 3 — Discuss the Approach

Explain:

- Brute Force
- Better Solution
- Optimal Solution

Interviewers value your reasoning process.

---

## Step 4 — Write Clean Code

Focus on:

- Meaningful variable names
- Small helper methods
- Readable logic
- Proper indentation

---

## Step 5 — Analyze Complexity

Always discuss:

- Time Complexity
- Space Complexity

Example:

```
Time : O(n)

Space : O(1)
```

---

## Step 6 — Dry Run

Test with:

- Normal cases
- Edge cases
- Empty input
- Single element
- Large input

---

## Common Mistakes

❌ Jumping directly into coding

❌ Ignoring constraints

❌ Forgetting edge cases

❌ Poor variable names

❌ Not explaining the solution

---

## Interview Tips

✔ Think aloud

✔ Stay calm

✔ Clarify assumptions

✔ Optimize gradually

✔ Verify before submitting

---

Strong communication combined with pattern recognition often makes the difference between a good solution and a successful interview.
