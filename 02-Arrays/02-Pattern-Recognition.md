# 🧩 Pattern Recognition

---

# Why Pattern Recognition?

Most interview problems are solved by recognizing the underlying pattern rather than memorizing solutions.

---

# Pattern 1

## Simple Traversal

Traverse every element once.

Examples:

- Find Maximum
- Find Minimum
- Count Even Numbers

Complexity

```
O(n)
```

---

# Pattern 2

## Prefix Sum

Used for range sum queries.

Problems

- Range Sum Query
- Subarray Sum Equals K

---

# Pattern 3

## Sliding Window

Used for contiguous subarrays.

Problems

- Maximum Sum Subarray
- Longest Substring Without Repeating Characters

---

# Pattern 4

## Two Pointers

Useful for sorted arrays or searching from both ends.

Problems

- Two Sum II
- Container With Most Water

---

# Pattern 5

## Kadane's Algorithm

Find Maximum Subarray Sum.

Problem

Maximum Subarray

---

# Pattern 6

## Dutch National Flag

Sort 0s, 1s and 2s.

Problem

Sort Colors

---

# Pattern 7

## Merge Intervals

Used for overlapping intervals.

Problems

- Merge Intervals
- Insert Interval

---

# Pattern 8

## Binary Search on Arrays

Works on sorted arrays.

Problems

- Binary Search
- Search Insert Position

---

# Interview Tip

Before writing code, ask yourself:

- Is this a Sliding Window problem?
- Can HashMap help?
- Can Two Pointers solve this?
- Is Prefix Sum applicable?

Pattern recognition often leads directly to the optimal solution.
