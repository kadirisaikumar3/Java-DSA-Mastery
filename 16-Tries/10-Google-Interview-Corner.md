# 💼 Google Interview Corner

---

# What Google Evaluates

- Trie Structure
- Prefix Matching
- Efficient String Search
- DFS + Trie
- Time Complexity
- Space Complexity

---

# Common Follow-up Questions

- Why Trie instead of HashMap?
- Why is Trie faster for prefixes?
- Can Trie support deletion?
- How do you reduce Trie memory usage?
- Can Trie handle uppercase or Unicode characters?

---

# Edge Cases

- Empty String
- Duplicate Words
- Prefix Equals Whole Word
- Missing Prefix
- Very Long Strings

---

# Communication Tips

While solving:

- Explain the Trie node structure.
- Describe how characters are processed.
- Explain why the operation is O(L).
- Mention Time & Space Complexity.

---

# Google Advice

Don't just explain the algorithm.

Explain:

- Why Trie is better than brute force.
- How shared prefixes improve efficiency.
- Why each character is processed only once.

This demonstrates strong algorithmic reasoning.
