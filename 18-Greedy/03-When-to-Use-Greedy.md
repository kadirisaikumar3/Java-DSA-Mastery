# ✅ When to Use Greedy

---

# Greedy is Suitable When

- Local optimum leads to global optimum.
- Previous decisions never need to change.
- Problem satisfies Greedy Choice Property.
- Problem has Optimal Substructure.

---

# Examples

- Activity Selection
- Fractional Knapsack
- Huffman Coding
- Minimum Platforms
- Job Sequencing

---

# Greedy vs Dynamic Programming

Greedy

- Makes one decision
- Never revisits

Dynamic Programming

- Explores multiple possibilities
- Stores previous results

---

# Interview Tip

If choosing the current best option never hurts future decisions,

↓

Think **Greedy**.
