# 💼 Google Interview Corner

---

# What Google Evaluates

- Pointer Manipulation
- Algorithm Design
- Time Complexity
- Space Complexity
- Edge Case Handling
- Clean Code

---

# Common Follow-up Questions

- Why use a Dummy Node?
- Why Fast & Slow Pointer?
- Can this be done in one pass?
- Can space complexity be reduced?
- How do you handle empty lists?

---

# Edge Cases

- Empty List
- Single Node
- Two Nodes
- Even Length
- Odd Length
- Cycle Present
- Duplicate Values

---

# Communication Tips

While solving:

- Explain pointer movement.
- Draw the Linked List.
- Explain why pointers change.
- Mention Time & Space Complexity.

---

# Google Advice

Don't just write pointer updates.

Explain:

- Why each pointer exists.
- What happens after every iteration.
- Why the algorithm remains correct.

This demonstrates strong problem-solving and communication skills.
