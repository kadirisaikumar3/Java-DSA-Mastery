# Time and Space Complexity

Understanding complexity helps you choose the most efficient algorithm.

---

## Common Time Complexities

| Complexity | Example |
|------------|---------|
| O(1) | HashMap Lookup |
| O(log n) | Binary Search |
| O(n) | Linear Search |
| O(n log n) | Merge Sort |
| O(n²) | Bubble Sort |
| O(2ⁿ) | Backtracking |
| O(n!) | Permutations |

---

## Common Space Complexities

| Complexity | Example |
|------------|---------|
| O(1) | Two Pointers |
| O(log n) | Recursive Binary Search |
| O(n) | BFS Queue |
| O(n²) | DP Matrix |

---

## Golden Rule

Choose the simplest algorithm that satisfies the required constraints while keeping time and memory usage efficient.
