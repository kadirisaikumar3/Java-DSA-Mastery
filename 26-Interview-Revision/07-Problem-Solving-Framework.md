# Problem Solving Framework

A structured approach is more valuable than memorizing solutions.

---

## Step 1

Understand the problem.

Ask questions.

Clarify constraints.

---

## Step 2

Think about the brute-force solution.

Can it work?

Can it be optimized?

---

## Step 3

Identify the pattern.

Examples:

- Sliding Window
- Binary Search
- DFS
- Dynamic Programming

---

## Step 4

Choose the appropriate data structure.

Examples:

- HashMap
- Heap
- Queue
- Stack
- Graph

---

## Step 5

Write pseudocode.

Avoid jumping directly into coding.

---

## Step 6

Write clean code.

Readable code is easier to debug.

---

## Step 7

Analyze:

Time Complexity

Space Complexity

---

## Step 8

Dry Run

Test:

- Empty Input
- Single Element
- Large Input
- Duplicate Values

---

## Interview Formula

```
Understand

↓

Brute Force

↓

Optimize

↓

Code

↓

Test

↓

Explain
```

Following this framework consistently helps reduce mistakes and improves interview performance.
