# 🚀 Optimal Approach

---

# Idea

Build a Segment Tree once and answer all future queries efficiently.

---

# Advantages

- Fast Range Queries
- Fast Point Updates
- Supports Dynamic Arrays
- Easily Extendable for Lazy Propagation

---

# Complexity

Build

```
O(n)
```

Query

```
O(log n)
```

Update

```
O(log n)
```

---

# Why It Works

Only relevant tree nodes are visited during a query or update.

---

# Interview Tip

Segment Trees are ideal when both **queries** and **updates** are frequent.
