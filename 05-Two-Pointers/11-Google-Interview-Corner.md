# 💼 Google Interview Corner

---

# What Google Evaluates

- Problem Solving
- Communication
- Time Complexity
- Space Complexity
- Code Quality
- Edge Case Handling

---

# Common Follow-up Questions

- Why Two Pointers instead of nested loops?
- Can you reduce space complexity?
- Is sorting allowed?
- What if the array is not sorted?
- Can you solve it in one pass?

---

# Edge Cases

- Empty Array
- Single Element
- Duplicate Values
- Negative Numbers
- Already Sorted
- All Same Values

---

# Communication Tips

While solving:

- Explain the brute-force approach.
- Describe pointer movement.
- Justify correctness.
- State Time & Space Complexity.

---

# Google Advice

Don't just move pointers.

Explain **why** moving the left or right pointer cannot miss the correct answer.
