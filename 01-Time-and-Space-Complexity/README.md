> "Writing correct code is only half the job. Writing efficient code is what makes a great software engineer."

---

## 📚 Overview

Time and Space Complexity are the foundation of Data Structures and Algorithms.

Whenever you solve a problem, interviewers don't just evaluate whether your solution is correct—they also assess **how efficiently it performs**.

A solution that works for 10 elements but fails for 10 million elements is not considered scalable.

This chapter teaches you how to analyze algorithms, compare different approaches, and choose the most efficient solution.

# 🌍 Real-World Importance

Imagine Google Search.

Google processes billions of searches every day.

Suppose one search algorithm takes:

```
O(n²)
```

Instead of

```
O(log n)
```

The difference would cost millions of CPU operations every second.

Efficient algorithms save:

- CPU Time
- Memory
- Electricity
- Server Costs
- User Waiting Time

# 📈 Complexity Order (Best → Worst)

```
O(1)

↓

O(log n)

↓

O(n)

↓

O(n log n)

↓

O(n²)

↓

O(n³)

↓

O(2ⁿ)

↓

O(n!)
```
