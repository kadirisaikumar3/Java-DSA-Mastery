# 💼 Google Interview Corner

---

# What Google Evaluates

- Stack Pattern Recognition
- Time Complexity
- Space Complexity
- Edge Case Handling
- Clean Implementation
- Communication Skills

---

# Common Follow-up Questions

- Why use a Stack?
- Can this be solved without a Stack?
- Why Monotonic Stack?
- Can space complexity be improved?
- What are the edge cases?

---

# Edge Cases

- Empty Stack
- Single Element
- Duplicate Elements
- Increasing Order
- Decreasing Order
- Balanced/Unbalanced Parentheses

---

# Communication Tips

While solving:

- Explain why Stack is suitable.
- Describe push/pop operations.
- Explain why elements are removed.
- Mention Time & Space Complexity.

---

# Google Advice

Don't just write stack operations.

Explain:

- Why each element is pushed.
- Why each element is popped.
- Why every element is processed at most once.

This demonstrates a strong understanding of Stack algorithms.
